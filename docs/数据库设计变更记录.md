## V2 → V2.1 变更详情

### 一、新增接口配置表组（3张表）

| 序号 | 表名 | 说明 |
|------|------|------|
| 18 | interface_config | 接口配置主表 |
| 19 | interface_mapping_table | 映射表配置（支持主子表） |
| 20 | interface_field_mapping | 字段映射配置 |

### 二、设计目的

**业务场景**：医嘱数据同步时，接口同时返回医嘱主表和医嘱子表的主子结构数据。

**设计方案**：通过三层配置实现多表映射：
1. `interface_config`：接口基本信息
2. `interface_mapping_table`：配置目标表，支持主子表关联
3. `interface_field_mapping`：字段级别映射

### 三、关键设计

```
interface_config (接口配置)
  └── interface_mapping_table (映射表配置)
        ├── ORDER_MAIN (主表映射)
        │     └── interface_field_mapping (字段映射)
        └── ORDER_ITEM (子表映射)
              ├── parent_mapping_id = ORDER_MAIN.id
              ├── relation_field = "order_id"
              └── interface_field_mapping (字段映射)
```

### 四、前端界面调整

**当前界面问题**：字段映射只支持单表，无法配置主子表结构。

**需要调整的内容**：

| 调整项 | 当前设计 | 新设计 |
|--------|----------|--------|
| 映射结构 | 平铺列表 | 树形结构（主表→子表） |
| 目标表选择 | 单选 | 支持多表，可添加子表 |
| 数据路径配置 | 无 | 新增 `data_path` 配置 |
| 关联字段配置 | 无 | 新增 `relation_field` 配置 |

**界面调整方案**：

- id, patient_code, name, gender, age
- department_id, bed_number, diagnosis, admission_date

**V1字段**：

**变更原因**：

**V2**：
- 患者基本信息

## V2.2 增量同步变更详情

### 一、变更背景

**问题**：HIS业务数据量庞大，每次全量同步不现实。需要增量同步策略。

**解决方案**：
- 首次同步：拉取30天内的数据
- 后续同步：只拉取24小时内变更的数据

                      ▼

│  数据映射写入交班系统表                                   │
│  - 新记录：INSERT                                        │

                      ▼

#### 5.2 同步服务核心逻辑

```typescript
function calculateSyncTimeRange(config: InterfaceConfig): { startTime: Date; endTime: Date; syncType: string } {
  const now = new Date()
  const isFirstSync = config.isFirstSync === true || !config.lastSyncTime
  
  if (isFirstSync) {
    const days = config.firstSyncDays || 30
    const startTime = new Date(now.getTime() - days * 24 * 60 * 60 * 1000)
    return { startTime, endTime: now, syncType: '首次同步' }
  } else {
    const hours = config.incrementalSyncHours || 24
    const lastSync = config.lastSyncTime ? new Date(config.lastSyncTime) : new Date(now.getTime() - hours * 60 * 60 * 1000)
    const startTime = new Date(lastSync.getTime() - 1 * 60 * 60 * 1000)
    return { startTime, endTime: now, syncType: '增量同步' }
  }
}
```
