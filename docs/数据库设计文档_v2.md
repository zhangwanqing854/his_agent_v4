# 医护智能交接班系统 - 数据库设计文档 V2

## 1. 设计原则

### 1.1 核心原则

- **HIS数据同步**：患者、就诊、转科、诊断、医嘱等数据从HIS系统同步
- **交班业务独立**：交班报告基于同步数据生成，交班筛选有明确业务规则
- **患者与就诊分离**：患者只存身份信息，就诊信息独立成表

### 1.2 交班筛选规则

| 筛选条件 | 说明 |
|---------|------|
| 昨日新入科患者 | 通过转科记录判断，`transfer_time` 在昨日且转入本科室 |
| I级护理患者 | 就诊表中 `nurse_level = '一级'` |

**注意**：只有满足上述条件之一的患者才需要交班，不是所有患者都交班。

### 1.3 交班报告内容

- 患者基本信息（床号、姓名、性别、年龄）
- 诊断信息（主诊断）
- 前12小时临时医嘱

## 3. HIS同步表组

### 3.1 患者表 (patient)

**说明**：存储患者基本身份信息，从HIS系统同步。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | BIGINT | 是 | 主键，自增 |
| patient_no | VARCHAR(50) | 是 | 患者编号（住院号），唯一 |
| name | VARCHAR(50) | 是 | 姓名 |
| gender | VARCHAR(10) | 是 | 性别（男/女） |
| birth_date | DATE | 否 | 出生日期 |
| age | INT | 否 | 年龄（冗余字段，便于查询） |

| phone | VARCHAR(20) | 否 | 联系电话 |
| address | VARCHAR(200) | 否 | 家庭住址 |
| emergency_contact | VARCHAR(50) | 否 | 紧急联系人 |
| emergency_phone | VARCHAR(20) | 否 | 紧急联系电话 |

### 4.3 前12小时临时医嘱处理

**设计说明**：不单独设计交班医嘱表，前12小时临时医嘱在生成交班报告时查询，合并写入 `handover_patient.current_condition` 字段。

**生成交班报告时**：
1. 查询前12小时临时医嘱
2. 将医嘱内容格式化
3. 与患者当前病情描述合并，写入 `current_condition` 字段

**current_condition 字段示例内容**：
```
【目前情况】
患者今日血糖波动较大，最高15.2mmol/L，需注意低血糖反应。

【前12小时临时医嘱】
1. 2026-04-01 08:00 血糖测定（急诊）
2. 2026-04-01 10:30 胰岛素注射液 8U 皮下注射
3. 2026-04-01 14:00 血糖测定
```

## 5. 交班筛选逻辑

### 5.1 筛选条件SQL

```sql
-- 查询需要交班的患者
SELECT DISTINCT v.id as visit_id, v.patient_id, v.bed_no, v.nurse_level,
       CASE 
           WHEN tr.id IS NOT NULL AND v.nurse_level = '一级' THEN 'BOTH'
           WHEN tr.id IS NOT NULL THEN 'NEW_ADMISSION'
           ELSE 'LEVEL1_NURSING'
       END as filter_reason
FROM visit v
LEFT JOIN transfer_record tr ON tr.visit_id = v.id 
    AND tr.to_dept_id = v.dept_id
    AND DATE(tr.transfer_time) = CURDATE() - INTERVAL 1 DAY  -- 昨日转入
WHERE v.dept_id = #{currentDeptId}
  AND v.patient_status = '在院'
  AND (
      DATE(tr.transfer_time) = CURDATE() - INTERVAL 1 DAY  -- 昨日新入科
      OR v.nurse_level = '一级'  -- I级护理
  );
```

### 5.2 获取前12小时临时医嘱SQL

```sql
-- 查询前12小时临时医嘱
SELECT oi.*, om.order_no, om.order_category
FROM order_item oi
JOIN order_main om ON oi.main_id = om.id
WHERE om.visit_id = #{visitId}
  AND om.order_type = '临时医嘱'  -- 临时医嘱
  AND oi.order_time >= NOW() - INTERVAL 12 HOUR  -- 前12小时
  AND om.status = '有效'
ORDER BY oi.order_time DESC;
```

### 5.3 获取主诊断SQL

```sql
-- 获取主诊断
SELECT di.diagnosis_name
FROM diagnosis_item di
JOIN diagnosis_main dm ON di.main_id = dm.id
WHERE dm.visit_id = #{visitId}
  AND di.is_main = 1  -- 主诊断
  AND dm.status = '有效'
ORDER BY dm.diagnosis_time DESC
LIMIT 1;
```

## 8. 关键设计说明

### 8.1 为什么要分离患者表和就诊表？

- **一对多关系**：一个患者可以多次住院，每次住院是独立的就诊记录
- **数据性质不同**：患者身份信息相对稳定，就诊信息动态变化
- **便于HIS同步**：HIS系统中患者信息和就诊信息本身就是分开的

### 8.2 为什么要设计转科记录表？

- **业务需要**：判断"昨日新入科"患者需要转科历史
- **完整记录**：记录患者完整的转科轨迹
- **查询效率**：独立表便于按时间、科室查询

### 8.3 为什么医嘱要区分长期/临时？

- **交班内容需要**：交班报告只展示前12小时临时医嘱
- **医嘱性质不同**：长期医嘱持续有效，临时医嘱执行一次
- **HIS标准设计**：符合HIS系统医嘱管理标准

### 8.4 filter_reason字段的作用

- **追溯原因**：记录患者为什么被选中交班
- **统计分析**：便于统计新入科、I级护理患者数量
- **审核需要**：交班报告需要展示筛选依据
