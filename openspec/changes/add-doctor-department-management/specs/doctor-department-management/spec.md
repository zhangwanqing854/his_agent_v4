## ADDED Requirements

### 需求:用户可以查看科室人员关系列表

系统必须提供科室人员关系列表页面，展示所有科室人员关系记录，支持分页和搜索。

#### 场景:成功加载列表
- **当** 用户进入系统设置页面的科室人员管理 Tab
- **那么** 系统显示科室人员关系列表，包含医护人员姓名、科室名称、是否主科室、创建时间

#### 场景:搜索过滤
- **当** 用户在搜索框输入医护人员姓名或科室名称
- **那么** 系统过滤列表，仅显示匹配的记录

#### 场景:分页浏览
- **当** 用户点击分页按钮
- **那么** 系统加载对应页的数据，每页显示10条记录

### 需求:用户可以新增科室人员关系

系统必须允许用户选择医护人员和科室，新增科室人员关系记录。

#### 场景:成功新增
- **当** 用户点击"新增"按钮，选择医护人员和科室，点击保存
- **那么** 系统创建新的科室人员关系记录，显示成功消息

#### 场景:医护人员不存在
- **当** 用户选择的医护人员在系统中不存在
- **那么** 系统提示"请先添加该医护人员"

#### 场景:科室不存在
- **当** 用户选择的科室在系统中不存在
- **那么** 系统提示"请先添加该科室"

#### 场景:重复关系
- **当** 用户尝试新增已存在的科室人员关系
- **那么** 系统提示"该关系已存在，请勿重复添加"

### 需求:用户可以编辑科室人员关系

系统必须允许用户修改科室人员关系的主科室标记。

#### 场景:成功编辑
- **当** 用户点击编辑按钮，修改是否主科室标记，点击保存
- **那么** 系统更新记录，显示成功消息

#### 场景:设置主科室
- **当** 用户将某关系标记为主科室，该医护人员已有其他主科室
- **那么** 系统自动将原主科室标记取消，仅保留当前为主科室

### 需求:用户可以删除科室人员关系

系统必须允许用户删除科室人员关系记录，删除前需确认。

#### 场景:成功删除
- **当** 用户点击删除按钮，确认删除
- **那么** 系统删除该记录，显示成功消息

#### 场景:取消删除
- **当** 用户点击删除按钮，取消确认
- **那么** 系统保留该记录，不做任何更改
