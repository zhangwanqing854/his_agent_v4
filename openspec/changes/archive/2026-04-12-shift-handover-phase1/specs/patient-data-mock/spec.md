## 新增需求

### 需求：患者基本信息管理

系统必须支持手动录入和编辑患者基本信息，包括床号、姓名、年龄、性别、入院日期、诊断列表、过敏史、手术史。

#### 场景：成功录入患者信息
- **当** 医生填写完整的患者信息表单并提交
- **那么** 系统保存患者信息并显示在患者列表中

#### 场景：患者信息验证
- **当** 医生提交患者信息时床号为空或姓名为空
- **那么** 系统拒绝保存并提示必填项缺失

#### 场景：患者信息编辑
- **当** 医生选择已存在的患者并修改信息后保存
- **那么** 系统更新患者信息并记录修改时间和修改人

---

### 需求：生命体征数据录入

系统必须支持录入患者生命体征数据（体温、脉搏、呼吸、血压、血氧），并支持按时间序列查看。

#### 场景：录入生命体征
- **当** 医生选择患者并填写生命体征数据
- **那么** 系统保存数据并关联到该患者

#### 场景：生命体征时间序列展示
- **当** 医生查看患者详情时
- **那么** 系统以时间轴形式展示最近 24 小时的生命体征数据

#### 场景：异常值高亮
- **当** 生命体征数据超出正常范围（体温>38.5°C 或<35°C，收缩压>180 或<90mmHg 等）
- **那么** 系统以红色高亮显示异常值

---

### 需求：检验检查结果录入

系统必须支持录入检验检查异常结果，包括项目名称、结果值、单位、参考范围、异常标志（高/低）。

#### 场景：录入检验结果
- **当** 医生填写检验项目、结果值和参考范围
- **那么** 系统自动判断并标记异常标志（HIGH/LOW/NORMAL）

#### 场景：批量导入检验结果
- **当** 医生上传 CSV 格式的检验结果文件
- **那么** 系统解析文件并批量导入检验结果，显示导入成功/失败统计

---

### 需求：患者列表筛选

系统必须支持按科室、床号范围、患者状态（在院/出院）筛选患者列表。

#### 场景：按科室筛选
- **当** 医生选择科室后
- **那么** 系统仅显示该科室的患者

#### 场景：按床号范围筛选
- **当** 医生输入床号范围（如 1-10）
- **那么** 系统仅显示该范围内的患者
