## 新增需求

### 需求：科室交班发起

系统必须支持医生以科室为维度发起交班流程，一次交接科室所有患者，生成或编辑交班报告，选择接班人。

**设计原则**：一次交班 = 整个科室所有患者，减少医生操作次数（30名患者 = 1次交班，而非30次）。

#### 场景：成功发起科室交班
- **当** 医生选择科室、查看科室患者列表、确认交班报告内容、选择接班人后点击"发起交班"
- **那么** 系统创建科室交班记录（shift_handover），为每个患者创建交班明细（handover_patient），状态设为 PENDING，并通知接班人

#### 场景：科室患者自动加载
- **当** 医生选择科室后
- **那么** 系统自动加载该科室所有在院患者列表，并为每个患者生成 SBAR 报告草稿

#### 场景：重点患者标记
- **当** 医生在患者列表中标记某患者为重点患者
- **那么** 系统在该患者的交班明细中设置 is_critical = true，并可填写 critical_reason

#### 场景：交班报告必填项检查
- **当** 医生发起交班但重点患者的 SBAR 报告内容为空
- **那么** 系统拒绝发起并提示"重点患者交班报告内容不能为空"

#### 场景：选择同一人作为交班和接班
- **当** 医生选择自己作为接班人
- **那么** 系统拒绝并提示"接班人不能与交班人相同"

---

### 需求：科室交班报告编辑

系统必须支持医生在发起交班前编辑 AI 生成的交班报告，支持对科室所有患者批量处理，重点患者详细编辑。

#### 场景：查看科室患者报告摘要
- **当** 医生进入交班发起页面
- **那么** 系统显示科室患者列表，每个患者显示简化的 SBAR 摘要

#### 场景：编辑单个患者报告
- **当** 医生点击某患者的"详细编辑"按钮
- **那么** 系统展开该患者的完整 SBAR 编辑表单

#### 场景：批量确认报告
- **当** 医生点击"全部采纳 AI 生成"按钮
- **那么** 系统将所有患者的 AI 生成报告设为最终版本

#### 场景：恢复 AI 原始生成
- **当** 医生点击某患者的"恢复 AI 生成"按钮
- **那么** 系统用 AI 原始生成内容替换该患者的当前编辑内容，并提示确认

---

### 需求：科室接班确认

系统必须支持接班人查看科室交班记录，一次确认接收整个科室的患者，或退回补充。

#### 场景：成功确认科室接班
- **当** 接班人查看科室交班记录、浏览所有患者交班信息后点击"确认接班"
- **那么** 系统更新科室交班状态为 COMPLETED，记录接班时间和签名，完成整个科室交接

#### 场景：查看科室交班详情
- **当** 接班人点击"查看"科室交班记录
- **那么** 系统显示科室统计摘要（总患者数、重点患者数、待办事项数）和详细患者列表

#### 场景：退回补充
- **当** 接班人点击"退回补充"并填写退回原因
- **那么** 系统更新状态为 DRAFT，通知交班人，并显示退回原因

#### 场景：查看科室交班历史
- **当** 医生查看某科室的历史交班记录
- **那么** 系统按时间倒序展示该科室所有已完成的交班记录

---

### 需求：科室交班状态流转

系统必须遵循 DRAFT → PENDING → TRANSFERRING → COMPLETED 的状态流转规则，禁止跳过状态。状态流转针对科室交班主表（shift_handover）。

#### 场景：正常状态流转
- **当** 科室交班记录依次经历 AI 生成（DRAFT）、医生确认（PENDING）、发起交班（TRANSFERRING）、接班确认（COMPLETED）
- **那么** 系统按状态机规则流转状态

#### 场景：非法状态跳转
- **当** 系统尝试将 DRAFT 状态直接变更为 COMPLETED
- **那么** 系统拒绝并抛出状态流转异常

---

### 需求：待办事项关联

系统必须支持在科室交班中添加待办事项，每个待办事项关联到具体患者。

#### 场景：添加患者关联待办
- **当** 医生在交班发起页面添加待办事项并选择关联患者
- **那么** 系统创建待办记录（handover_todo），关联到科室交班和具体患者

#### 场景：待办事项优先级
- **当** 医生设置待办事项优先级（高/中/低）
- **那么** 系统按优先级排序显示待办列表

#### 场景：接班后待办提醒
- **当** 接班人确认接班后
- **那么** 系统将所有未完成待办事项推送至接班人的待办列表

---

### 需求：交班记录版本管理

系统必须为每次交班记录创建版本快照，支持查看历史版本对比。

#### 场景：创建新版本
- **当** 交班记录被修改后保存
- **那么** 系统创建新版本，保留旧版本内容

#### 场景：版本对比
- **当** 医生选择两个版本进行对比
- **那么** 系统高亮显示两个版本之间的差异内容
