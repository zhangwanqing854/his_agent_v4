## 新增需求

### 需求：待办事项自动提取

系统必须从 AI 生成的交班报告中自动提取待办事项，包括待执行医嘱、待完成检查、会诊安排等。

#### 场景：成功提取待办事项
- **当** AI 生成报告后
- **那么** 系统解析 Recommendation 部分，提取待办事项列表，包含事项内容、截止时间、优先级

#### 场景：手动添加待办事项
- **当** 医生在交班报告中手动添加待办事项
- **那么** 系统识别并添加到待办事项列表

---

### 需求：待办事项展示

系统必须在交班报告页面和独立的待办事项页面展示待办事项，支持按优先级、截止时间排序。

#### 场景：按优先级排序
- **当** 医生选择"按优先级"排序
- **那么** 系统按高→中→低优先级显示待办事项

#### 场景：按截止时间排序
- **当** 医生选择"按截止时间"排序
- **那么** 系统按截止时间从早到晚显示待办事项

---

### 需求：待办事项推送

系统必须支持将待办事项推送给接班人，Phase 1 使用站内消息推送。

#### 场景：交班时推送待办事项
- **当** 交班记录状态变更为 TRANSFERRING
- **那么** 系统向接班人发送站内消息，包含待办事项列表

#### 场景：待办事项提醒
- **当** 待办事项截止时间前 1 小时仍未标记完成
- **那么** 系统发送提醒消息给责任人

---

### 需求：待办事项状态管理

系统必须支持待办事项状态管理，包括待执行、执行中、已完成、已取消。

#### 场景：标记待办事项完成
- **当** 接班人点击"标记完成"
- **那么** 系统更新状态为已完成，记录完成时间和完成人

#### 场景：取消待办事项
- **当** 医生取消某个待办事项并填写取消原因
- **那么** 系统更新状态为已取消，记录取消原因

---

### 需求：待办事项追踪

系统必须支持追踪待办事项的执行情况，支持查看历史待办事项。

#### 场景：查看待办事项历史
- **当** 医生查看某患者的待办事项历史
- **那么** 系统展示该患者所有历史待办事项及执行状态

#### 场景：待办事项统计
- **当** 科室主任查看科室待办事项统计
- **那么** 系统展示待办事项总数、完成率、超时率等指标
