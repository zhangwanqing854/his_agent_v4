# 用户管理规格说明

## 能力描述

系统必须提供用户管理功能，支持RBAC权限模型，实现用户账号的全生命周期管理，并与HIS系统同步的医生信息自动关联。

---

## 用户故事

### US-1: 管理员登录
**作为** 系统管理员
**我希望** 使用 admin/admin 账号登录系统
**以便** 管理系统用户和权限

**验收标准：**
- **当** 管理员输入用户名 admin 和密码 admin
- **那么** 系统验证通过，返回登录成功
- **并且** 返回用户信息和权限列表
- **并且** is_super_admin 字段为 true

---

### US-2: HIS同步医生自动创建用户
**作为** 系统管理员
**我希望** HIS同步新医生时自动创建系统账号
**以便** 医生能立即使用系统

**验收标准：**
- **当** HIS系统同步新的医生信息
- **那么** 系统自动创建用户账号
- **并且** 用户名为医生的工号
- **并且** 初始密码为医生的工号
- **并且** 角色自动设为"普通医生"
- **并且** his_staff_id 关联到同步的医生记录

---

### US-3: 查看用户列表
**作为** 系统管理员
**我希望** 查看系统中所有用户列表
**以便** 了解系统用户情况

**验收标准：**
- **当** 管理员访问用户管理页面
- **那么** 显示所有用户列表
- **并且** 显示用户名、姓名、角色、关联医生、状态、创建时间
- **并且** 支持按用户名、角色、状态筛选
- **并且** 支持分页

---

### US-4: 新增用户
**作为** 系统管理员
**我希望** 手动创建新的系统用户
**以便** 给新员工分配系统账号

**验收标准：**
- **当** 管理员点击"新增用户"按钮
- **那么** 显示用户编辑弹窗
- **当** 填写用户名、密码、选择角色后保存
- **那么** 用户创建成功
- **并且** 用户名不能与已有用户重复
- **并且** 可选择关联HIS医生

---

### US-5: 编辑用户
**作为** 系统管理员
**我希望** 修改用户信息
**以便** 更新用户的角色或关联医生

**验收标准：**
- **当** 管理员点击用户的"编辑"按钮
- **那么** 显示用户编辑弹窗，预填当前信息
- **当** 修改信息后保存
- **那么** 用户信息更新成功
- **并且** 用户名不可修改

---

### US-6: 删除用户
**作为** 系统管理员
**我希望** 删除不再使用的用户账号
**以便** 清理系统中的无效账号

**验收标准：**
- **当** 管理员点击用户的"删除"按钮
- **那么** 显示删除确认弹窗
- **当** 确认删除
- **那么** 用户被删除
- **但是** 超级管理员账号（admin）不可删除
- **并且** 不能删除自己

---

### US-7: 启用/禁用用户
**作为** 系统管理员
**我希望** 启用或禁用用户账号
**以便** 控制用户是否能够登录系统

**验收标准：**
- **当** 管理员点击"禁用"按钮
- **那么** 用户状态变为禁用
- **并且** 该用户无法登录系统
- **但是** 超级管理员账号（admin）不可禁用
- **并且** 不能禁用自己

---

### US-8: 重置密码
**作为** 系统管理员
**我希望** 重置用户的密码
**以便** 用户忘记密码时能恢复访问

**验收标准：**
- **当** 管理员点击"重置密码"按钮
- **那么** 显示重置密码弹窗
- **当** 输入新密码后确认
- **那么** 用户密码更新成功
- **并且** 用户下次登录使用新密码

---

### US-9: 角色管理
**作为** 系统管理员
**我希望** 管理系统角色和权限
**以便** 灵活配置用户权限

**验收标准：**
- **当** 管理员访问角色管理页面
- **那么** 显示所有角色列表
- **并且** 可查看每个角色拥有的权限
- **当** 编辑角色的权限
- **那么** 角色权限更新成功
- **并且** 拥有该角色的用户权限立即生效

---

### US-10: 权限校验
**作为** 系统
**我需要** 校验用户是否有权限访问某功能
**以便** 控制功能访问

**验收标准：**
- **当** 用户访问某功能
- **如果** 用户 is_super_admin = true
- **那么** 直接放行，不检查权限
- **如果** 用户 is_super_admin = false
- **那么** 检查用户角色是否拥有对应权限
- **如果** 无权限
- **那么** 返回 403 Forbidden

---

### US-11: 普通用户修改密码
**作为** 普通用户
**我希望** 修改自己的登录密码
**以便** 保护账号安全

**验收标准：**
- **当** 用户访问个人设置页面
- **那么** 可修改自己的密码
- **当** 输入旧密码、新密码后保存
- **那么** 密码修改成功
- **并且** 下次登录使用新密码

---

## 业务规则

### BR-1: 用户名唯一
用户名在系统中必须唯一，不允许重复。

### BR-2: 超级管理员保护
用户名为 admin 且 is_super_admin = 1 的账号：
- 不可删除
- 不可禁用
- 密码可修改

### BR-3: 自我操作限制
用户不能：
- 删除自己的账号
- 禁用自己的账号

### BR-4: HIS同步医生默认权限
HIS同步过来的新医生，自动获得"普通医生"角色，拥有以下权限：
- 交班管理
- 患者管理
- 待办事项
- 统计分析

### BR-5: 密码加密存储
所有密码必须使用 BCrypt 加密后存储，不可明文存储。

## 数据约束

### 用户表约束

| 字段 | 约束 |
|------|------|
| username | 长度 1-50，字母数字下划线，唯一 |
| password | 加密后长度 ≤ 255 |
| role_id | 外键，必须存在 |

### 角色表约束

| 字段 | 约束 |
|------|------|
| name | 长度 1-50，必填 |
| code | 长度 1-50，字母数字下划线，唯一 |

### 权限表约束

| 字段 | 约束 |
|------|------|
| code | 长度 1-50，字母数字下划线，唯一 |
| name | 长度 1-50，必填 |
