## 新增需求

### 求:用户可以查看科室排班列表
系统必须允许用户查看当前科室的排班计划列表，按月份展示。

#### 场景:查看排班列表
- **当** 用户进入排班管理页面
- **那么** 系统显示当前科室所有排班计划，按月份排序

#### 场景:按月份筛选
- **当** 用户选择特定月份
- **那么** 系统显示该月份的排班详情

### 需求:用户可以创建月度排班计划
系统必须允许用户创建新的月度排班计划。

#### 场景:创建排班计划
- **当** 用户点击"创建排班"按钮并选择月份
- **那么** 系统创建空白排班计划，状态为"草稿"

#### 场景:创建已存在月份排班
- **当** 用户尝试创建已存在排班的月份
- **那么** 系统提示"该月份排班已存在"并禁止创建

### 需求:用户可以编辑排班明细
系统必须允许用户为排班计划添加和修改每日值班人员。

#### 场景:添加值班人员
- **当** 用户选择某一天并从科室人员列表中选择值班人员
- **那么** 系统将该人员设置为该天的值班人员

#### 场景:更换值班人员
- **当** 用户为某一天选择新的值班人员
- **那么** 系统更新该天的值班人员为新选择的人员

#### 场景:清空值班人员
- **当** 用户清除某一天的值班人员设置
- **那么** 系统将该天标记为空（无值班人员）

#### 场景:添加备注
- **当** 用户为排班明细输入备注信息
- **那么** 系统保存备注内容

### 需求:用户可以删除排班计划
系统必须允许用户删除排班计划。

#### 场景:删除草稿排班
- **当** 用户删除状态为"草稿"的排班计划
- **那么** 系统删除排班及其所有明细

#### 场景:删除已发布排班
- **当** 用户删除状态为"已发布"的排班计划
- **那么** 系统提示确认后删除排班及其所有明细

### 需求:用户可以发布排班计划
系统必须允许用户将排班计划状态从"草稿"改为"已发布"。

#### 场景:发布排班
- **当** 用户点击"发布"按钮
- **那么** 系统将排班状态改为"已发布"，该排班对所有科室人员可见

### 需求:排班以卡片形式展示
系统必须以卡片形式展示排班信息，便于直观浏览。

#### 场景:卡片展示排班
- **当** 用户查看排班详情
- **那么** 系统以卡片形式展示每日排班信息，包含日期和值班人员姓名

#### 场景:卡片布局
- **当** 排班卡片渲染
- **那么** 系统按日期顺序排列卡片，每行显示一周（7天）

#### 场景:空值班显示
- **当** 某一天无值班人员
- **那么** 系统在该卡片位置显示空白或"待安排"提示

### 需求:用户可以打印排班卡片
系统必须支持排班卡片的打印输出，一张 A4 纸打印两个月排班。

#### 场景:打印双月排班
- **当** 用户点击"打印"按钮
- **那么** 系统生成打印预览，横向布局显示两个月排班卡片

#### 场景:A4纸张适配
- **当** 打印输出生成
- **那么** 系统确保两个月排班卡片适配一张 A4 纸（210mm × 297mm）

#### 场景:打印样式
- **当** 打印预览显示
- **那么** 系统隐藏页面导航和其他非打印元素，仅显示排班卡片内容

### 需求:排班人员必须是当前科室工作人员
系统必须限制排班人员范围，仅允许从当前科室工作人员中选择。

#### 场景:获取可排班人员
- **当** 用户添加值班人员
- **那么** 系统仅显示当前科室的工作人员列表

#### 场景:跨科室人员限制
- **当** 用户尝试选择非本科室人员
- **那么** 系统禁止该操作，人员列表不显示其他科室人员

### 求:排班数据必须按科室隔离
系统必须确保排班数据按科室隔离，用户只能查看和管理自己科室的排班。

#### 场景:科室数据隔离
- **当** 用户查询排班列表
- **那么** 系统仅返回用户当前科室的排班数据

#### 场景:跨科室访问禁止
- **当** 用户尝试访问其他科室排班
- **那么** 系统拒绝访问并提示"无权限"

### 需求:用户可以快速生成排班
系统必须支持基于人员排序自动生成排班，减少手动排班工作量。

#### 场景:设置人员排序
- **当** 用户进入快速生成对话框
- **那么** 系统显示科室人员列表，用户可拖拽调整排班顺序

#### 场景:保存人员排序
- **当** 用户调整人员排序后点击保存
- **那么** 系统保存排序到科室配置，后续排班可复用

#### 场景:首次排班选择起点
- **当** 科室首次进行排班（无上月数据）
- **那么** 系统提示用户选择从排序第几位开始排班

#### 场景:非首次排班自动接续
- **当** 科室已有上月排班数据
- **那么** 系统自动从上月月末值班人员的下一位开始继续轮换

#### 场景:连续轮换算法
- **当** 用户设置排序为[张三, 李四, 王五]，上月月末排到张三
- **那么** 本月第一天排到李四，第二天排到王五，第三天排到张三，依此循环

#### 场景:指定生成范围
- **当** 用户在快速生成对话框设置开始和结束日期
- **那么** 系统仅在该日期范围内生成排班

#### 场景:周末正常排班
- **当** 生成范围包含周末
- **那么** 系统对周末同样进行排班轮换

#### 场景:生成后手动微调
- **当** 快速生成完成
- **那么** 用户可点击任意日期手动修改值班人员
