## 为什么

当前排班功能自动从科室工作人员（doctor_department 表）获取可排班人员，但实际排班中并非所有科室工作人员都需要参与值班（如行政人员、实习生等）。缺少专门维护值班人员名单的功能，导致排班人员列表过于冗长，快速生成排班时需要手动筛选，效率低下。

## 变更内容

新增科室值班人员管理功能，支持：

1. **值班人员名单维护** - 科室可单独维护参与排班的值班人员名单，与科室工作人员列表独立管理
2. **人员增删** - 从科室工作人员中选择需要参与值班的人员，添加到值班人员表；也可移除不需要参与值班的人员
3. **排班数据源切换** - 创建排班和快速生成排班时，从值班人员表获取人员列表，不再从科室工作人员表自动获取

## 功能 (Capabilities)

### 新增功能

- `duty-staff-config`: 科室值班人员配置功能，用于维护科室参与排班的值班人员名单

### 修改功能

- `department-scheduling`: 排班人员获取方式变更，从值班人员表获取可排班人员，不再从 doctor_department 表自动获取

## 影响

| 影响范围 | 说明 |
|---------|------|
| 前端页面 | 新增值班人员配置对话框组件（DutyStaffConfigDialog.vue），修改排班管理页面集成配置入口 |
| 后端 API | 新增值班人员配置 Controller、Service；修改 SchedulingService.getSchedulableStaff() 数据来源 |
| 数据库 | 新增科室值班人员表（department_duty_staff） |
| 现有接口 | `/api/scheduling/staff` 接口数据来源变更 |

**与 HIS/LIS/PACS 系统对接点**: 无。值班人员信息来源于系统内部的工作人员数据，不涉及外部系统同步。

## 非目标

- 不支持跨科室配置值班人员
- 不实现值班人员自动推荐算法
- 不支持值班人员模板导入导出
- 不修改现有的工作人员管理（HisStaff）功能
