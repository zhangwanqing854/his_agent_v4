## 修改需求

### 需求:排班人员必须是当前科室工作人员
系统必须限制排班人员范围，仅允许从当前科室已配置的值班人员中选择。

#### 场景:获取可排班人员
- **当** 用户添加值班人员
- **那么** 系统仅显示当前科室已配置的值班人员列表

#### 场景:值班人员表为空时
- **当** 科室值班人员配置表为空
- **那么** 系统显示提示"请先配置值班人员"，排班人员列表为空

#### 场景:跨科室人员限制
- **当** 用户尝试选择非本科室人员
- **那么** 系统禁止该操作，人员列表不显示其他科室人员

### 需求:用户可以快速生成排班
系统必须支持基于值班人员排序自动生成排班，减少手动排班工作量。

#### 场景:设置人员排序
- **当** 用户进入快速生成对话框
- **那么** 系统显示科室已配置的值班人员列表，用户可拖拽调整排班顺序

#### 场景:保存人员排序
- **当** 用户调整人员排序后点击保存
- **那么** 系统保存排序到科室配置，后续排班可复用

#### 场景:首次排班选择起点
- **当** 科室首次进行排班（无上月数据）
- **那么** 系统提示用户选择从排序第几位开始排班

#### 场景:非首次排班自动接续
- **当** 科室已有上月排班数据
- **那么** 系统自动从上月月末值班人员的下一位开始继续轮换

#### 场景:连续轮换算法
- **当** 用户设置排序为[张三, 李四, 王五]，上月月末排到张三
- **那么** 本月第一天排到李四，第二天排到王五，第三天排到张三，依此循环

#### 场景:指定生成范围
- **当** 用户在快速生成对话框设置开始和结束日期
- **那么** 系统仅在该日期范围内生成排班

#### 场景:周末正常排班
- **当** 生成范围包含周末
- **那么** 系统对周末同样进行排班轮换

#### 场景:生成后手动微调
- **当** 快速生成完成
- **那么** 用户可点击任意日期手动修改值班人员

## 移除需求

无 - 此变更不移除现有需求。
