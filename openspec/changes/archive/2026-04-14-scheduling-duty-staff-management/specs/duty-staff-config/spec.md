## 新增需求

### 需求:科室可以维护值班人员名单
系统必须允许科室单独维护参与排班的值班人员名单，与科室工作人员列表独立管理。

#### 场景:查看值班人员列表
- **当** 用户进入值班人员配置页面
- **那么** 系统显示当前科室已配置的值班人员列表

#### 场景:无值班人员配置
- **当** 科室首次进入值班人员配置页面且无配置数据
- **那么** 系统显示空列表，并提示用户"请添加值班人员"

### 需求:科室可以从工作人员中添加值班人员
系统必须允许科室从当前科室工作人员中选择人员添加到值班人员名单。

#### 场景:显示可添加人员列表
- **当** 用户点击"添加值班人员"按钮
- **那么** 系统显示当前科室工作人员列表（已添加为值班人员的除外）

#### 场景:添加单个值班人员
- **当** 用户从工作人员列表中选择人员并确认添加
- **那么** 系统将该人员添加到值班人员名单

#### 场景:批量添加值班人员
- **当** 用户选择多个工作人员并确认批量添加
- **那么** 系统将所有选中人员添加到值班人员名单

#### 场景:添加已存在的值班人员
- **当** 用户尝试添加已在值班人员名单中的人员
- **那么** 系统提示"该人员已在值班人员名单中"并禁止重复添加

### 需求:科室可以移除值班人员
系统必须允许科室从值班人员名单中移除不需要参与值班的人员。

#### 场景:移除值班人员
- **当** 用户选择某值班人员并点击"移除"按钮
- **那么** 系统将该人员从值班人员名单中移除

#### 场景:移除最后一个值班人员
- **当** 用户尝试移除值班人员名单中唯一的人员
- **那么** 系统提示确认"移除后将无法排班"，用户确认后执行移除

### 需求:科室可以调整值班人员排序
系统必须允许科室调整值班人员的显示顺序，用于快速生成排班时的人员轮换顺序。

#### 场景:调整人员顺序
- **当** 用户通过拖拽调整值班人员顺序
- **那么** 系统保存新的顺序配置

#### 场景:一键初始化值班人员
- **当** 科室值班人员名单为空时用户点击"一键初始化"
- **那么** 系统将所有科室工作人员导入值班人员名单，并按默认顺序排列

### 需求:值班人员配置必须按科室隔离
系统必须确保值班人员配置按科室隔离，用户只能查看和管理自己科室的值班人员。

#### 场景:科室数据隔离
- **当** 用户查询值班人员配置
- **那么** 系统仅返回用户当前科室的值班人员数据

#### 场景:跨科室操作禁止
- **当** 用户尝试操作其他科室的值班人员配置
- **那么** 系统拒绝操作并提示"无权限"
