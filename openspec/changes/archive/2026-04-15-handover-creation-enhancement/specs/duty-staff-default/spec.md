## 新增需求

### 需求:系统必须默认选择当天值班人员为接班医生
系统必须在发起交班页面加载时自动选择当天值班人员作为接班医生的默认值。

#### 场景:有排班数据时自动选择
- **当** 用户进入发起交班页面且当天有排班数据
- **那么** 系统自动选择当天值班人员作为接班医生默认值

#### 场景:无排班数据时显示提示
- **当** 用户进入发起交班页面且当天无排班数据
- **那么** 系统显示"无排班数据，请手动选择接班医生"提示，接班医生字段为空

#### 场景:排班未发布时显示提示
- **当** 用户进入发起交班页面且当天排班状态为草稿
- **那么** 系统显示"排班未发布，请手动选择接班医生"提示，接班医生字段为空

### 需求:用户必须可以修改接班医生
系统必须允许用户修改接班医生，修改范围限定为当前科室的工作人员。

#### 场景:手动修改接班医生
- **当** 用户点击接班医生下拉框
- **那么** 系统显示当前科室工作人员列表供用户选择

#### 场景:接班医生必须是当前科室人员
- **当** 用户选择接班医生
- **那么** 系统仅允许选择当前科室的工作人员

### 需求:值班人员必须从排班表获取
系统必须从排班明细表（scheduling_detail）获取当天值班人员信息。

#### 场景:查询当天值班人员
- **当** 系统获取接班医生默认值
- **那么** 系统查询排班表中当天日期对应的值班人员

#### 场景:排班按科室隔离
- **当** 系统查询值班人员
- **那么** 系统仅查询用户当前科室的排班数据
