## 修改需求

### 需求:班次默认值必须为夜班
系统必须将发起交班页面的班次默认值设置为"夜班"，而非"白班"。

#### 场景:默认选择夜班
- **当** 用户进入发起交班页面
- **那么** 系统默认选中"夜班"班次

#### 场景:用户可以切换班次
- **当** 用户点击班次选项
- **那么** 系统允许用户切换为"白班"或保持"夜班"

### 需求:交班患者列表必须按条件筛选
系统必须只显示符合条件的患者：在院患者、24小时内入院的患者和I级护理的患者。

#### 场景:仅显示在院患者
- **当** 患者fg_ip='Y'（在院标识）且discharge_datetime大于当前时间或为空
- **那么** 该患者符合在院条件

#### 场景:显示24小时内入院患者
- **当** 患者符合在院条件且24小时内入院
- **那么** 该患者出现在交班患者列表中

#### 场景:显示I级护理患者
- **当** 患者符合在院条件且护理等级为I级（nurse_level_code='01'）
- **那么** 该患者出现在交班患者列表中

#### 场景:不显示不在院患者
- **当** 患者fg_ip='N'或discharge_datetime小于当前时间
- **那么** 该患者不出现在交班患者列表中

#### 场景:不显示不符合筛选条件的患者
- **当** 患者不符合24小时内入院或I级护理条件
- **那么** 该患者不出现在交班患者列表中

#### 场景:显示筛选原因
- **当** 患者出现在交班列表中
- **那么** 系统显示筛选原因标签（如"24小时内新入院"、"一级护理"、"24小时内新入院+一级护理"）

### 需求:患者统计数据必须自动计算
系统必须自动计算患者统计数据，用户无需手动输入。

#### 场景:统计数据自动显示
- **当** 用户进入发起交班页面
- **那么** 系统自动显示计算好的统计数据（入院、转出、出院、转入）

#### 场景:统计数据不可手动编辑
- **当** 用户查看统计数据
- **那么** 统计数据为只读状态，不可手动修改
