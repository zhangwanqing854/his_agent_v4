## 新增需求

### 需求:系统必须自动计算患者统计数据
系统必须在发起交班页面加载时自动计算并显示当前科室的患者统计数据，包括患者总数、入院人数、转出人数、出院人数、转入人数。

#### 场景:显示患者总数
- **当** 用户进入发起交班页面
- **那么** 系统显示当前科室在院患者总数

#### 场景:显示24小时内入院人数
- **当** 用户进入发起交班页面
- **那么** 系统自动计算并显示24小时内入院的患者人数

#### 场景:显示24小时内转出人数
- **当** 用户进入发起交班页面
- **那么** 系统自动计算并显示24小时内从当前科室转出的患者人数

#### 场景:显示24小时内出院人数
- **当** 用户进入发起交班页面
- **那么** 系统自动计算并显示24小时内出院的患者人数

#### 场景:显示24小时内转入人数
- **当** 用户进入发起交班页面
- **那么** 系统自动计算并显示24小时内转入当前科室的患者人数

### 需求:系统必须不显示死亡手术病危人数
系统必须在发起交班页面中隐藏死亡人数、手术人数、病危人数的统计和显示。

#### 场景:不显示死亡人数
- **当** 用户进入发起交班页面
- **那么** 系统不显示死亡人数统计项

#### 场景:不显示手术人数
- **当** 用户进入发起交班页面
- **那么** 系统不显示手术人数统计项

#### 场景:不显示病危人数
- **当** 用户进入发起交班页面
- **那么** 系统不显示病危人数统计项

### 需求:统计数据必须按科室隔离
系统必须确保患者统计数据按科室隔离，仅统计用户当前登录科室的患者数据。

#### 场景:科室数据隔离
- **当** 用户查询患者统计数据
- **那么** 系统仅返回用户当前科室的统计数据

#### 场景:切换科室后数据更新
- **当** 用户切换当前登录科室
- **那么** 系统重新计算并显示新科室的患者统计数据
