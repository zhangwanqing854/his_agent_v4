## 新增需求

### 需求:系统必须自动填充临时医嘱到目前情况
系统必须在加载交班患者列表时，自动将患者24小时内的临时医嘱信息填充到"目前情况"字段。

#### 场景:有临时医嘱时自动填充
- **当** 患者在24小时内有临时医嘱
- **那么** 系统将临时医嘱信息自动填充到该患者的"目前情况"字段

#### 场景:无临时医嘱时字段为空
- **当** 患者在24小时内无临时医嘱
- **那么** 该患者的"目前情况"字段为空，用户可手动输入

#### 场景:用户可以编辑目前情况
- **当** 系统自动填充临时医嘱后
- **那么** 用户仍可手动编辑修改"目前情况"内容

### 需求:临时医嘱必须从医嘱表获取
系统必须从医嘱表（order_main）获取24小时内的临时医嘱，筛选条件为临时标识（order_type='N'）和医嘱时间在24小时内。

#### 场景:筛选临时医嘱
- **当** 系统查询患者临时医嘱
- **那么** 系统仅查询order_type='N'的医嘱记录

#### 场景:筛选24小时内医嘱
- **当** 系统查询患者临时医嘱
- **那么** 系统仅查询start_time在24小时内的医嘱记录

### 需求:临时医嘱显示格式必须简洁
系统必须以简洁格式显示临时医嘱，包括医嘱类别和关键内容，不做完整详情展示。

#### 场景:显示医嘱类别和内容
- **当** 系统填充临时医嘱
- **那么** 显示格式为"医嘱类别: 医嘱内容"，多条医嘱以换行分隔

#### 场景:医嘱内容过长时截断
- **当** 单条医嘱内容超过100字符
- **那么** 系统截断显示前100字符并添加省略号
