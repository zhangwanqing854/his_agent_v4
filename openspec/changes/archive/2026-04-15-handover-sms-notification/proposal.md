## 为什么

交班提交后，接班医生可能无法及时获知有新的交班任务等待处理，导致交班流程延迟。通过短信通知可以确保接班医生第一时间收到提醒，提高交班效率和及时性。

## 变更内容

1. 新增短信发送服务（可扩展接口设计，支持接入阿里云/腾讯云等短信平台）
2. 交班提交（`submitHandover`）时自动发送短信给接班医生
3. 短信内容包含：交班科室、交班医生、患者数量、交班时间
4. 电话号码从 `his_staff.phone` 字段获取
5. 新增前端短信配置管理界面，支持管理员配置短信平台参数

## 功能 (Capabilities)

### 新增功能
- `sms-notification`: 短信通知服务，支持在交班提交时自动发送短信给接班医生，短信平台接口可扩展
- `sms-config`: 短信配置管理，支持在前端界面配置短信平台参数（启用/禁用、平台选择、API密钥等）

### 修改功能
- `handover-submission`: 交班提交流程需集成短信发送逻辑（在状态从 DRAFT → PENDING 时触发）

## 影响

### 后端
- 新增 `SmsService` 接口及实现类
- 新增 `SmsConfig` 实体类及数据库表（存储配置参数）
- 新增 `SmsConfigController` 提供 API 接口
- 修改 `HandoverService.submitHandover()` 方法
- 新增短信发送日志表 `sms_log`

### 前端
- 新增短信配置管理页面 `SmsConfigPage.vue`
- 新增短信配置 API 调用
- 在系统设置中添加短信配置入口

### 外部依赖
- 需接入短信服务平台（阿里云短信/腾讯云短信/医院现有平台）

## 非目标

- 不实现短信模板管理功能
- 不实现短信发送失败后的自动重试机制（第一版本）
- 不实现批量短信发送
- 不实现短信发送历史查看界面（第一版本）
