## 新增需求

### 需求:交班提交必须触发短信通知

系统在交班提交（状态从 DRAFT 变为 PENDING）时必须触发短信通知发送给接班医生。

#### 场景:提交交班触发短信
- **当** 交班医生点击"提交交班"
- **那么** 系统将交班状态从 DRAFT 变为 PENDING，并触发短信发送给接班医生

#### 场景:短信发送不阻塞提交
- **当** 交班提交触发短信发送
- **那么** 交班提交立即返回成功，短信异步发送

#### 场景:无接班医生不发送短信
- **当** 交班 `to_doctor_id` 为空
- **那么** 系统不发送短信（此情况交班提交本身也会失败）

### 需求:短信发送必须关联交班记录

短信发送必须关联对应的交班记录（`handover_id`），便于追溯。

#### 场景:日志关联交班
- **当** 系统记录短信发送日志
- **那么** `sms_log.handover_id` 必须等于对应的交班 ID
