## 新增需求

### 需求:系统必须提供短信配置管理功能

系统必须提供短信配置管理功能，允许管理员在前端界面配置短信平台参数。

#### 场景:查看配置
- **当** 管理员访问短信配置页面
- **那么** 系统显示当前短信配置（敏感字段脱敏显示）

#### 场景:修改配置
- **当** 管理员修改配置并点击保存
- **那么** 系统保存配置到数据库，无需重启服务即可生效

### 需求:系统必须支持启用禁用短信功能

系统必须支持通过配置界面启用或禁用短信通知功能。

#### 场景:禁用短信
- **当** 管理员将"启用短信通知"开关设置为关闭
- **那么** 系统不再发送任何短信通知

#### 场景:启用短信
- **当** 管理员将"启用短信通知"开关设置为开启
- **那么** 系统恢复短信发送功能

### 需求:系统必须支持配置多个短信平台

系统必须支持配置阿里云、腾讯云等多个短信平台，管理员可选择使用哪个平台。

#### 场景:选择阿里云
- **当** 管理员选择"阿里云"作为短信平台
- **那么** 系统显示阿里云配置项（AccessKey ID、AccessKey Secret、签名、模板Code）

#### 场景:选择腾讯云
- **当** 管理员选择"腾讯云"作为短信平台
- **那么** 系统显示腾讯云配置项

### 需求:系统必须对敏感配置字段进行脱敏显示

系统必须在前端显示敏感配置字段（如密钥）时进行脱敏处理，仅显示部分内容。

#### 场景:脱敏显示密钥
- **当** 配置字段标记为敏感（is_sensitive=true）
- **那么** 前端显示时仅显示前4位字符，其余用星号替代（如 `ak****xx`）

#### 场景:显示完整密钥
- **当** 用户点击"显示"按钮
- **那么** 系统显示该字段的完整内容

### 需求:系统必须提供测试发送功能

系统必须提供测试发送功能，允许管理员验证配置是否正确。

#### 场景:测试发送成功
- **当** 管理员点击"测试发送"并输入测试手机号
- **那么** 系统使用当前配置发送测试短信

#### 场景:测试发送失败提示
- **当** 测试发送失败
- **那么** 系统显示错误信息，提示配置可能有问题

### 需求:短信配置必须存储在数据库

短信配置必须存储在 `sms_config` 数据库表，支持动态修改无需重启服务。

#### 场景:配置持久化
- **当** 管理员保存配置
- **那么** 配置保存到 `sms_config` 表，记录更新时间和更新人

### 需求:系统必须提供短信配置 API

系统必须提供短信配置 API 接口供前端调用。

#### 场景:获取配置 API
- **当** 前端调用 GET /api/sms-config
- **那么** 系统返回当前配置（敏感字段已脱敏）

#### 场景:更新配置 API
- **当** 前端调用 PUT /api/sms-config
- **那么** 系统更新配置并返回成功响应
