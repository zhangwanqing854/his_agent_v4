## 新增需求

### 需求:系统必须支持交班详情弹窗查看

用户点击交班管理列表的"查看"按钮时，系统必须弹出交班详情弹窗，显示交班基本信息和患者列表。

#### 场景:点击查看按钮弹出详情弹窗
- **当** 用户在交班管理列表点击某条交班记录的"查看"按钮
- **那么** 系统弹出交班详情弹窗，显示该交班的基本信息

#### 场景:弹窗显示交班基本信息
- **当** 交班详情弹窗打开
- **那么** 弹窗显示科室名称、交班医生、接班医生、交班日期、班次、状态等信息

#### 场景:弹窗显示患者列表
- **当** 交班详情弹窗打开且数据加载完成
- **那么** 弹窗显示该交班关联的患者列表，包含床号、姓名、诊断、目前情况等字段

#### 场景:数据加载失败提示错误
- **当** 加载交班患者数据失败
- **那么** 系统显示错误提示消息，弹窗仍然保持打开状态

### 需求:查看按钮必须正确响应点击事件

查看按钮的事件处理必须不受任何事件修饰符干扰，确保点击能正确触发 handleView 函数。

#### 场景:按钮点击正确触发处理函数
- **当** 用户点击查看按钮
- **那么** handleView 函数被正确调用，接收对应的交班行数据

#### 场景:弹窗显示加载状态
- **当** 点击查看按钮后正在加载患者数据
- **那么** 弹窗显示加载状态指示器

### 需求:已完成交班支持打印报告

对于状态为已完成的交班，详情弹窗必须提供打印交班报告的功能。

#### 场景:已完成交班显示打印按钮
- **当** 交班状态为 COMPLETED
- **那么** 弹窗底部显示"打印交班报告"按钮

#### 场景:点击打印打开报告预览
- **当** 用户点击打印交班报告按钮
- **那么** 系统打开报告预览弹窗，显示格式化的交班报告
