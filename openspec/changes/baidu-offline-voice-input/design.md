## 上下文

当前 VoiceInputDialog.vue 使用浏览器原生 Web Speech API：
```javascript
const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition
recognition.lang = 'zh-CN'
```

此 API 依赖云端服务，无法离线运行。需要替换为百度离线语音 SDK。

## 目标 / 非目标

**目标：**
- 支持离线环境下的语音识别
- 保持现有语音录入交互流程（单人录入、批量录入、二维码扫码）
- 中文医疗术语识别准确率满足需求

**非目标：**
- 不修改语音解析逻辑（voiceParser.ts）
- 不修改患者匹配逻辑（patientMatcher.ts）
- 不修改语音录入的 UI 组件结构
- 不实现语音指令控制功能

## 决策

### 决策 1：语音识别 SDK 选择

**选择：** 百度离线语音 SDK（DuerOS 离线语音识别）

**考虑方案：**

| 方案 | 离线支持 | 中文准确率 | 成本 | 集成复杂度 |
|------|---------|-----------|------|-----------|
| Web Speech API | ❌ 不支持 | 中等 | 免费 | 低 |
| 百度离线语音 | ✓ 支持 | 高 | 付费 | 中 |
| 讯飞离线语音 | ✓ 支持 | 高 | 高成本 | 高 |
| Vosk（开源） | ✓ 支持 | 中等 | 免费 | 高 |

**理由：**
- 百度离线语音 SDK 中文识别准确率高，适合医疗场景
- 相比讯飞，百度离线 SDK 成本更低
- 百度提供 Web 端 SDK，与现有 Vue 项目兼容性好

### 冺策 2：SDK 集成方式

**选择：** 前端直接集成百度离线语音 Web SDK

**方案：**
```
┌─────────────────────────────────────────────────────────────────┐
│                     语音识别架构                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  VoiceInputDialog.vue                                          │
│  │                                                              │
│  │  startSpeechRecognition()                                   │
│  │      ↓                                                       │
│  │  ┌─────────────────────────────────────────────────────┐    │
│  │  │  百度离线语音 Web SDK                                  │    │
│  │  │                                                       │    │
│  │  │  • 录音：MediaRecorder API                            │    │
│  │  │  • 识别：本地模型（下载到浏览器）                      │    │
│  │  │  • 输出：识别文本                                      │    │
│  │  └─────────────────────────────────────────────────────┘    │
│  │                                                              │
│  │  transcriptText.value = 识别结果                            │
│  │                                                              │
│  ↓                                                              │
│  voiceParser.ts → patientMatcher.ts → 回写患者信息              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 冺策 3：模型文件部署

**选择：** 模型文件部署在后端，前端首次使用时下载缓存

**方案：**
- 后端提供模型文件下载接口：`/api/voice/model`
- 前端首次调用语音识别时检查本地缓存，如无则下载
- 模型文件约 50-100MB，下载后缓存到浏览器 IndexedDB

## 风险 / 权衡

- **风险：模型文件较大，首次下载耗时** → 提示用户下载进度，支持离线包预装
- **风险：百度 SDK License 过期** → 设置 License 有效期提醒，提前续费
- **风险：离线识别准确率不如在线** → 可保留在线模式作为备选方案
- **权衡：增加前端包体积** → 模型文件单独加载，不影响主应用体积
