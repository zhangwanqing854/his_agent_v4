## 为什么

当前系统使用浏览器原生 Web Speech API 进行语音识别，依赖 Google/Apple 云端服务，**无法离线运行**。医院内网环境可能无法访问外部网络，导致语音录入功能失效。需要替换为支持离线运行的语音识别方案。

## 变更内容

- 将语音识别从 Web Speech API 替换为百度离线语音 SDK
- 支持离线环境下的语音录入功能
- 保持现有语音录入交互流程不变（单人录入、批量录入、二维码扫码）

## 功能

### 新增功能

- `offline-voice-recognition`: 离线语音识别功能，使用百度离线语音 SDK

### 修改功能

- `handover-creation`: 语音录入实现从 Web Speech API 改为百度离线语音 SDK

## 影响

- 前端：VoiceInputDialog.vue - 替换语音识别实现
- 前端：新增百度离线语音 SDK 依赖
- 后端：可能需要新增语音识别结果处理接口（可选）
- 部署：需要配置百度离线语音 SDK 的 License 和模型文件
- 非目标：不修改语音解析逻辑、患者匹配逻辑、UI 交互流程
