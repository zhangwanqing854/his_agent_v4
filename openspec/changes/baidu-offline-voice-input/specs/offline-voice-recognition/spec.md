## ADDED Requirements

### 需求:系统必须支持离线语音识别

系统必须使用百度离线语音 SDK 进行语音识别，确保在医院内网离线环境下仍可正常使用语音录入功能。

#### 场景:离线环境语音识别成功
- **当** 用户在无网络连接环境下点击语音录入按钮
- **那么** 系统使用本地语音模型进行识别，返回识别文本

#### 场景:首次使用下载语音模型
- **当** 用户首次使用语音录入功能且本地无语音模型缓存
- **那么** 系统从后端下载语音模型文件，显示下载进度，下载完成后自动开始语音识别

### 需求:语音识别必须支持中文医疗术语

系统必须使用中文语音识别模型，支持医疗专业术语的准确识别。

#### 场景:医疗术语识别准确
- **当** 用户说出"血压"、"血糖"、"静脉注射"等医疗术语
- **那么** 系统正确识别并返回相应文本

### 需求:语音识别必须实时转写

系统必须在用户说话过程中实时显示识别结果，支持中间结果展示。

#### 场景:实时转写显示
- **当** 用户正在说话
- **那么** 系统实时显示已识别的文本内容，中间结果用灰色字体显示

#### 场景:识别结果稳定显示
- **当** 语音识别完成一段内容
- **那么** 系统将稳定的识别结果转为黑色字体显示
