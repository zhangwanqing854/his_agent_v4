## 新增需求

### 需求:批量语音录入模式
系统必须提供批量语音录入模式，允许医护人员通过一次语音输入同时录入多个患者的病情信息。

#### 场景:切换批量模式
- **当** 用户点击语音录入按钮并选择批量模式
- **那么** 系统显示批量语音录入界面，提示用户可以连续录入多个患者

#### 场景:批量录入流程
- **当** 用户在批量模式下开始录音
- **那么** 系统实时显示语音转写内容，支持用户随时暂停或停止录音

### 需求:语音内容解析
系统必须能够从语音转写文本中解析出患者标识（床号、姓名）和病情描述。

#### 场景:标准格式解析
- **当** 用户语音内容为"2床张三，病情稳定。5床李四，今日出院"
- **那么** 系统解析出两条记录：(床号2,姓名张三,病情稳定) 和 (床号5,姓名李四,今日出院)

#### 场景:非标准格式解析
- **当** 用户语音内容表述不规范（如仅说床号未说姓名）
- **那么** 系统尽可能解析床号信息，并在匹配结果中标记为需手动确认

### 需求:患者匹配
系统必须将解析出的床号/姓名与当前科室患者列表进行匹配，找到对应患者记录。

#### 场景:精确匹配
- **当** 解析出的床号和姓名与患者列表中的患者完全匹配
- **那么** 系统自动将病情描述关联到该患者

#### 场景:同名患者处理
- **当** 存在同名患者导致匹配不确定
- **那么** 系统在预览界面显示所有匹配候选，让用户手动选择

#### 场景:未匹配处理
- **当** 解析出的床号/姓名在患者列表中未找到匹配
- **那么** 系统在预览界面标记为未匹配，用户可手动选择患者

### 需求:匹配结果预览
系统必须在填充患者病情前显示匹配结果预览界面，供用户确认和修正。

#### 场景:预览界面显示
- **当** 语音录音停止并完成解析匹配
- **那么** 系统显示预览界面，列出所有解析条目及其匹配的患者

#### 场景:手动修正匹配
- **当** 用户在预览界面点击未匹配或匹配错误的条目
- **那么** 系统弹出患者选择下拉框，用户可手动选择正确患者

#### 场景:确认批量填充
- **当** 用户在预览界面点击确认按钮
- **那么** 系统将所有病情描述批量填充到对应患者的病情描述字段

### 需求:录音控制
系统必须支持录音过程中的暂停、继续、取消操作。

#### 场景:暂停录音
- **当** 用户在录音过程中点击暂停按钮
- **那么** 系统暂停录音，保留已转写内容，用户可继续录音

#### 场景:取消录音
- **当** 用户点击取消按钮
- **那么** 系统终止录音，丢弃已转写内容，返回上一界面
