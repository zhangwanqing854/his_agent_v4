## ADDED Requirements

### 需求:超级管理员标识可显示和修改
系统必须在用户管理界面显示超级管理员标识，并允许授权用户修改该标识。

#### 场景:显示超管标识
- **当** 用户列表加载完成
- **那么** 超管用户在用户名旁显示"超管"标签

#### 场景:超管可修改其他用户超管状态
- **当** 超管用户编辑非超管用户
- **那么** 显示"超级管理员"开关控件，可开启或关闭

#### 场景:非超管不可见超管开关
- **当** 非超管用户编辑任何用户
- **那么** 不显示"超级管理员"开关控件

#### 场景:超管不能修改自己
- **当** 超管用户编辑自己的账号
- **那么** "超级管理员"开关控件显示为禁用状态，提示"不可修改自己"

### 需求:最后一个超管保护
系统必须禁止移除最后一个超级管理员。

#### 场景:移除最后一个超管被拒绝
- **当** 超管用户尝试将某个用户的超管状态改为false
- **那么** 系统检查当前超管数量，如果只有1个，拒绝修改并返回错误消息"不能移除最后一个超级管理员"

### 需求:超管修改操作审计
系统必须记录超管标识修改的操作日志。

#### 场景:修改成功记录日志
- **当** 超管用户成功修改其他用户的超管状态
- **那么** 系统记录操作者ID、目标用户ID、修改前值、修改后值、操作时间
