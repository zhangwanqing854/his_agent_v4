## 新增需求

### 需求:患者行语音按钮
患者列表每行必须提供语音录入按钮，允许用户直接对该患者进行语音录入。

#### 场景:显示语音按钮
- **当** 用户进入发起交接班页面
- **那么** 每个患者行的操作列显示语音录入按钮（麦克风图标）

#### 场景:点击语音按钮
- **当** 用户点击某患者的语音录入按钮
- **那么** 系统打开语音录入对话框，并传入该患者信息，默认为单人模式

### 需求:录音后编辑界面
单人模式下录音停止后必须显示可编辑的文本界面，允许用户修改识别内容后再确认回写。

#### 场景:录音停止进入编辑
- **当** 用户停止录音
- **那么** 系统停止语音识别，显示编辑界面，文本框中显示识别内容

#### 场景:编辑识别内容
- **当** 编辑界面显示识别内容
- **那么** 用户可以在文本框中修改、删除或补充内容

#### 场景:确认并回写
- **当** 用户点击"确认"按钮
- **那么** 系统将编辑后的内容回写到该患者的"目前情况"字段

#### 场景:取消编辑
- **当** 用户点击"取消"按钮
- **那么** 系统关闭对话框，不回写任何内容

### 需求:保留编辑功能
用户必须能够在回写后继续修改或补充内容。

#### 场景:修改回写内容
- **当** 语音内容已回写到"目前情况"
- **那么** 用户可以点击患者表格字段进行手动修改
