## 新增需求

### 需求:外部触发入口
语音录入组件必须支持从外部传入患者信息，实现针对特定患者的单人语音录入。

#### 场景:传入患者信息
- **当** 外部组件调用语音录入对话框并传入患者对象
- **那么** 对话框默认为单人模式，录音目标锁定为该患者

#### 场景:自动回写
- **当** 传入患者信息且语音录入完成
- **那么** 系统直接将识别内容回写到传入患者的"目前情况"，不执行患者匹配逻辑

## 修改需求

### 需求:模式切换
语音录入组件必须支持单人模式和批量模式切换，默认为单人模式。

#### 场景:切换模式
- **当** 用户打开语音录入对话框
- **那么** 系统显示模式切换按钮（单人/批量），默认选中单人模式

#### 场景:单人模式
- **当** 用户选择单人模式并点击录音
- **那么** 系统执行现有的单人语音录入流程，录音内容直接填充到当前选中患者或传入患者

#### 场景:批量模式
- **当** 用户选择批量模式并点击录音
- **那么** 系统执行批量语音录入流程，录音停止后显示匹配预览界面

#### 场景:外部传入时锁定单人模式
- **当** 外部组件传入患者信息打开对话框
- **那么** 系统自动切换到单人模式并隐藏模式切换按钮，防止用户误切换到批量模式
