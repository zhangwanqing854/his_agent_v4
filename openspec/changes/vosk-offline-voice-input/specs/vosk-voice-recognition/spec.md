## 新增需求

### 需求:系统必须提供WebSocket语音识别服务

系统必须提供 WebSocket 端点用于实时语音识别，支持前端通过 WebSocket 连接发送音频数据并接收识别结果。

#### 场景:WebSocket连接成功
- **当** 前端发起 WebSocket 连接到 `/ws/voice`
- **那么** 连接成功建立，系统准备接收音频数据

#### 场景:WebSocket断开重连
- **当** WebSocket 连接意外断开
- **那么** 前端自动尝试重新连接，最多重试3次

### 需求:系统必须支持实时流式语音识别

系统必须支持实时流式语音识别，边接收音频边返回识别结果，支持临时结果和最终结果。

#### 场景:实时返回临时结果
- **当** 前端发送音频数据流
- **那么** 系统实时返回临时识别结果（partial），前端可实时显示

#### 场景:返回最终识别结果
- **当** 用户结束录音或发送结束信号
- **那么** 系统返回最终识别结果（final），包含完整文本

### 需求:系统必须支持离线运行

语音识别服务必须支持在无外部网络连接的内网环境下运行，不依赖任何云端服务。

#### 场景:内网环境语音识别
- **当** 系统部署在医院内网环境，无法访问外网
- **那么** 语音识别功能正常运行，用户可正常使用语音录入

### 需求:音频格式必须为PCM 16kHz

前端发送的音频数据必须为 PCM 格式，采样率 16kHz，16bit，单声道，以 base64 编码传输。

#### 场景:音频格式正确识别成功
- **当** 前端发送符合格式要求的 PCM 音频数据
- **那么** 系统正确识别并返回识别结果

#### 场景:音频格式错误拒绝处理
- **当** 前端发送不符合格式要求的音频数据
- **那么** 系统返回错误信息，提示音频格式不正确
