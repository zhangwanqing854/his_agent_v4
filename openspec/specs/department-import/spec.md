## 新增需求

### 需求:系统必须提供科室信息CSV导入入口

系统设置页面系统配置Tab必须新增"科室信息导入"功能卡片，包含导入按钮和CSV模板下载链接。

#### 场景:用户进入系统设置页面
- **当** 用户进入系统设置页面并点击"系统配置"Tab
- **那么** 系统显示"科室信息导入"卡片，包含"导入科室信息"按钮和"下载模板"链接

#### 场景:用户点击下载模板
- **当** 用户点击"下载模板"链接
- **那么** 系统下载`docs/data/科室信息.csv`模板文件

### 需求:系统必须校验上传的CSV文件格式

系统必须校验上传的CSV文件包含必需的列头（CODE, NAME, ID_DEP），且文件大小不超过限制。

#### 场景:上传标准格式CSV文件
- **当** 用户上传包含CODE、NAME、ID_DEP三列的CSV文件
- **那么** 系统通过校验，显示数据预览表格（前10行）

#### 场景:上传缺少必需列的CSV文件
- **当** 用户上传缺少CODE或NAME列的CSV文件
- **那么** 系统拒绝文件并提示"CSV文件缺少必需列：CODE, NAME"

#### 场景:上传超大CSV文件
- **当** 用户上传超过1000行的CSV文件
- **那么** 系统拒绝文件并提示"单次导入最多支持1000条数据，请分批导入"

### 需求:系统必须复用HIS科室信息同步的字段映射配置

系统必须通过查询`interface_config`表获取`config_code='HIS_DEPT_SYNC'`的映射配置，动态映射CSV字段到department表字段。

#### 场景:查询映射配置成功
- **当** 系统解析CSV文件后查询`HIS_DEPT_SYNC`的映射配置
- **那么** 系统获取`interface_field_mapping`记录，映射CODE→code, NAME→name

#### 场景:映射配置不存在
- **当** 系统查询`HIS_DEPT_SYNC`配置但记录不存在
- **那么** 系统记录错误日志，提示"未找到科室信息映射配置，请联系管理员"

### 需求:系统必须根据科室编码检测重复数据

系统必须根据department.code字段检测重复数据：若code已存在则更新记录，否则新增记录。

#### 场景:导入新科室信息
- **当** CSV中某行的CODE在department表中不存在
- **那么** 系统新增一条department记录，code和name取自CSV

#### 场景:导入已存在的科室信息
- **当** CSV中某行的CODE在department表中已存在
- **那么** 系统更新该department记录的name字段（code保持不变）

#### 场景:导入科室信息name为空
- **当** CSV中某行的NAME列为空值
- **那么** 系统跳过该行，记录到错误列表，提示"第N行：科室名称不能为空"

### 霹求:系统必须返回导入统计结果

系统必须返回包含总数、新增数、更新数、跳过数、失败数的导入统计结果，并提供错误详情。

#### 场景:导入成功无错误
- **当** 所有CSV数据行导入成功（无跳过或失败）
- **那么** 系统返回统计结果：totalCount=总行数，insertCount=新增数，updateCount=更新数，skipCount=0，failCount=0，errors=[]

#### 场景:导入部分失败
- **当** 部分CSV数据行导入失败（如必填字段为空）
- **那么** 系统返回统计结果包含failCount和errors列表，errors包含行号和错误原因

### 需求:系统必须提供导入进度反馈

系统必须在导入过程中显示进度状态，并在完成后展示结果摘要。

#### 场景:导入进行中
- **当** 后端正在处理CSV数据导入
- **那么** 前端显示进度条和"正在导入..."提示

#### 场景:导入完成
- **当** 后端返回导入结果
- **那么** 前端关闭进度条，展示结果摘要弹窗：成功数、更新数、失败数，并提供"查看详情"按钮

#### 场景:用户查看导入详情
- **当** 用户点击"查看详情"按钮
- **那么** 系统展示完整的errors列表，包含行号和错误原因

### 需求:系统必须提供导入对话框交互

系统必须通过对话框引导用户完成上传、预览、确认导入的流程。

#### 场景:用户点击导入按钮
- **当** 用户点击"导入科室信息"按钮
- **那么** 系统打开导入对话框，包含文件上传组件

#### 场景:用户上传文件后预览数据
- **当** 用户上传符合格式的CSV文件
- **那么** 对话框显示数据预览表格（前10行），并提供"确认导入"和"取消"按钮

#### 场景:用户确认导入
- **当** 用户点击"确认导入"按钮
- **那么** 系统发送POST请求到`/api/department/import`，显示进度状态

#### 场景:用户取消导入
- **当** 用户点击"取消"按钮或关闭对话框
- **那么** 系统清除已上传文件，关闭对话框
