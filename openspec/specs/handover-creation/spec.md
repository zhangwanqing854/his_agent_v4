## 新增需求

### 需求:创建交班必须自动生成编号

系统在创建交班时必须调用编号生成服务，自动为交班分配唯一编号。

#### 场景:创建交班自动编号
- **当** 用户发起交班
- **那么** 系统自动生成编号并设置到交班记录

#### 场景:编号生成失败处理
- **当** 编号生成失败（如唯一约束冲突）
- **那么** 系统重试生成新编号，最多重试3次

#### 场景:编号生成超限报错
- **当** 编号重试超过3次仍失败
- **那么** 系统返回错误，交班创建失败

### 需求:交班DTO必须包含编号字段

API 返回的交班数据必须包含 `handoverNo` 字段。

#### 场景:API返回编号
- **当** 前端请求交班列表或详情
- **那么** 返回数据包含 `handoverNo` 字段

### 需求:单人语音录入必须回写到需观察项字段

用户通过患者列表行的麦克风按钮进行单人语音录入时，识别内容必须回写到该患者的 `observationItems`（需观察项）字段。

#### 场景:单人语音录入成功回写
- **当** 用户点击患者行的麦克风按钮并完成语音录入
- **那么** 系统将识别内容写入该患者的 `observationItems` 字段

#### 场景:单人语音录入可编辑确认
- **当** 用户在编辑界面确认修改后的内容
- **那么** 系统将修改后的内容写入该患者的 `observationItems` 字段

### 需求:批量语音录入必须回写到需观察项字段

用户通过批量语音录入功能进行多患者录入时，匹配成功的内容必须回写到对应患者的 `observationItems`（需观察项）字段。

#### 场景:批量语音录入自动匹配回写
- **当** 用户完成批量语音录入且系统自动匹配患者成功
- **那么** 系统将识别内容写入匹配患者的 `observationItems` 字段

#### 场景:批量语音录入手动选择回写
- **当** 用户手动选择匹配患者后确认
- **那么** 系统将识别内容写入所选患者的 `observationItems` 字段
