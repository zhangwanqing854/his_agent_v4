## 新增需求

### 需求:创建交班必须自动生成编号

系统在创建交班时必须调用编号生成服务，自动为交班分配唯一编号。

#### 场景:创建交班自动编号
- **当** 用户发起交班
- **那么** 系统自动生成编号并设置到交班记录

#### 场景:编号生成失败处理
- **当** 编号生成失败（如唯一约束冲突）
- **那么** 系统重试生成新编号，最多重试3次

#### 场景:编号生成超限报错
- **当** 编号重试超过3次仍失败
- **那么** 系统返回错误，交班创建失败

### 需求:交班DTO必须包含编号字段

API 返回的交班数据必须包含 `handoverNo` 字段。

#### 场景:API返回编号
- **当** 前端请求交班列表或详情
- **那么** 返回数据包含 `handoverNo` 字段
