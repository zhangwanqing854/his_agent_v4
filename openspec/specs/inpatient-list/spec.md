## ADDED Requirements

### 需求:系统必须提供科室在院患者列表

系统必须提供科室在院患者列表功能，从 visit 表获取 `patient_status = '在院'` 的患者数据，按当前登录科室过滤。

#### 场景:查看在院患者列表
- **当** 用户进入科室患者页面
- **那么** 系统显示当前科室所有在院患者列表，包含床号、姓名、护理等级、入院时间

#### 场景:按科室过滤患者
- **当** 用户切换科室
- **那么** 患者列表自动刷新为新科室的在院患者

### 需求:系统必须支持患者筛选功能

系统必须支持按床号、姓名、护理等级筛选在院患者。

#### 场景:按床号筛选
- **当** 用户输入床号筛选条件
- **那么** 患者列表仅显示匹配床号的患者

#### 场景:按姓名筛选
- **当** 用户输入姓名筛选条件
- **那么** 患者列表仅显示姓名包含搜索关键字的患者

#### 场景:按护理等级筛选
- **当** 用户选择护理等级（特级、一级、二级、三级）
- **那么** 患者列表仅显示对应护理等级的患者

### 需求:页面标题必须显示为科室患者

页面标题必须显示为"科室患者"，并在标题旁显示当前科室名称。

#### 场景:显示科室名称
- **当** 用户进入科室患者页面
- **那么** 页面标题显示为"科室患者 神经外科病房"（当前科室名称）
