## ADDED Requirements

### 需求:用户管理页面必须支持分页显示

用户管理页面必须显示分页组件，允许用户按页浏览用户列表，每页可选择显示数量。

#### 场景:默认分页显示
- **当** 用户进入用户管理页面
- **那么** 页面底部显示分页组件，默认每页10条数据

#### 场景:切换页码
- **当** 用户点击分页组件的页码按钮
- **那么** 列表显示对应页码的用户数据

#### 场景:调整每页数量
- **当** 用户选择每页显示数量（10/20/50/100）
- **那么** 列表重新计算分页并显示相应数量的数据

### 需求:人员管理页面必须支持分页显示

人员管理页面必须显示分页组件，允许用户按页浏览人员列表。

#### 场景:默认分页显示
- **当** 用户进入人员管理页面
- **那么** 页面底部显示分页组件，默认每页10条数据

#### 场景:分页导航
- **当** 用户点击下一页或上一页按钮
- **那么** 列表显示对应页的人员数据

### 需求:科室管理页面必须支持分页显示

科室管理页面必须显示分页组件，允许用户按页浏览科室列表。

#### 场景:默认分页显示
- **当** 用户进入科室管理页面
- **那么** 页面底部显示分页组件，默认每页10条数据

#### 场景:快速跳转
- **当** 用户在分页输入框输入页码并回车
- **那么** 列表跳转到指定页码
