## 新增需求

### 需求:下拉选择框必须支持搜索筛选

所有数据量较大的下拉选择框必须支持搜索筛选功能，允许用户输入关键字快速定位目标选项。

#### 场景:科室选择搜索
- **当** 用户在科室下拉框中输入关键字
- **那么** 下拉框仅显示名称包含该关键字的科室选项

#### 场景:医生选择搜索
- **当** 用户在医生下拉框中输入关键字
- **那么** 下拉框仅显示姓名包含该关键字的医生选项

#### 场景:人员选择搜索
- **当** 用户在人员下拉框中输入关键字
- **那么** 下拉框仅显示姓名包含该关键字的人员选项

### 需求:固定选项下拉框无需搜索功能

选项数量固定且较少的下拉框（如状态选择、是/否选择）无需添加搜索功能。

#### 场景:状态选择无需搜索
- **当** 用户打开状态选择下拉框（启用/禁用）
- **那么** 下拉框显示全部固定选项，无需搜索输入框

### 需求:搜索输入框必须可清空

搜索输入框必须支持清空功能，用户清空后恢复显示全部选项。

#### 场景:清空搜索恢复全部选项
- **当** 用户清空搜索输入框内容
- **那么** 下拉框恢复显示全部选项
