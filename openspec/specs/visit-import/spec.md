## 新增需求

### 需求:用户可以导入就诊信息CSV文件

系统必须允许用户上传符合模板格式的 CSV 文件批量导入就诊信息到 visit 表。

#### 场景:成功上传CSV文件
- **当** 用户在系统设置页面数据导入 Tab 点击"导入就诊信息"按钮
- **那么** 系统显示就诊信息导入对话框，支持拖拽或点击上传 CSV 文件

#### 场景:文件格式校验失败
- **当** 用户上传非 CSV 格式文件
- **那么** 系统提示"仅支持 CSV 格式文件"并清空已上传文件

#### 场景:文件大小超限
- **当** 用户上传的 CSV 文件数据超过 1000 条
- **那么** 系统提示"单次导入最多支持 1000 条数据，请分批导入"并拒绝导入

### 需求:系统必须按HIS就诊信息同步的字段映射配置解析CSV

系统必须查询数据库 interface_config（config_code='HIS_VISIT_SYNC'）、interface_mapping_table、interface_field_mapping 表获取字段映射配置，按配置将 CSV 字段映射处理。

#### 场景:字段映射配置查询成功
- **当** 系统开始导入流程
- **那么** 系统查询 HIS_VISIT_SYNC 配置获取 15 个字段映射

#### 场景:字段映射配置缺失
- **当** HIS_VISIT_SYNC 配置不存在或字段映射为空
- **那么** 系统提示"未找到就诊信息映射配置，请联系管理员"并终止导入

### 需求:系统必须按visit_no唯一键判断重复数据

系统必须按 visit.visit_no 判断重复数据，存在则更新，不存在则插入。

#### 场景:新增就诊数据
- **当** CSV 行的 CODE_ENT 对应的 visit_no 在 visit 表不存在
- **那么** 系统插入新记录，insertCount +1

#### 场景:更新已有就诊数据
- **当** CSV 行的 CODE_ENT 对应的 visit_no 在 visit 表已存在
- **那么** 系统更新该记录的所有映射字段，updateCount +1

#### 场景:CODE_ENT为空
- **当** CSV 行的 CODE_ENT 字段为空或 null
- **那么** 系统跳过该行，记录错误"CODE_ENT 不能为空"到 errors 列表，failCount +1

### 需求:系统必须校验必需列不为空

系统必须校验 CSV 的 CODE_ENT、CODE_PAT、NAME_PAT、ID_PAT、CODE_DEP_PHY、NAME_DEP_PHY、SD_LEVEL_NUR、NAME_LEVEL_NUR、DT_ACPT 列不为空。

#### 场景:NAME_PAT为空
- **当** CSV 行的 NAME_PAT 字段为空或 null
- **那么** 系统跳过该行，记录错误"NAME_PAT 不能为空"到 errors 列表，failCount +1

#### 场景:DT_ACPT格式错误
- **当** CSV 行的 DT_ACPT 字段日期格式无法解析
- **那么** 系统跳过该行，记录错误"DT_ACPT 日期格式错误"到 errors 列表，failCount +1

#### 场景:可选字段缺失
- **当** CSV 行的 DT_END、CODE_ENTP 等可选字段缺失
- **那么** 系统将该字段映射的 visit 字段设为 null，继续导入

### 需求:系统必须正确处理DATE类型字段

系统必须正确解析 DT_ACPT（入院时间）和 DT_END（出院时间）日期字段，支持 yyyy-MM-dd HH:mm:ss、yyyy/M/d H:mm 等格式。

#### 场景:标准日期格式
- **当** DT_ACPT 字段值为 "2026-04-17 14:27"
- **那么** 系统解析为 LocalDateTime 并存入 admission_datetime 字段

#### 场景:斜杠分隔日期格式
- **当** DT_ACPT 字段值为 "2026/4/17 14:27"
- **那么** 系统正确解析为 LocalDateTime 并存入 admission_datetime 字段

#### 场景:DT_END为空
- **当** DT_END 字段值为空
- **那么** 系统将 discharge_datetime 设为 null，继续导入

### 需求:系统必须显示导入结果摘要

系统必须在导入完成后显示导入结果摘要，包括总计、新增、更新、跳过、失败数量，并允许查看错误详情。

#### 场景:导入成功
- **当** 导入完成且 failCount = 0
- **那么** 系统显示成功弹窗："导入成功：新增 X 条，更新 Y 条"

#### 场景:部分失败
- **当** 导入完成且 failCount > 0
- **那么** 系统显示警告弹窗："导入完成：失败 X 条"，提供"查看详情"按钮

#### 场景:查看错误详情
- **当** 用户点击"查看详情"按钮
- **那么** 系统显示 errors 列表，每条错误包含行号和错误原因

### 需求:系统必须提供CSV模板下载功能

系统必须提供就诊信息 CSV 模板下载功能，模板文件路径为 docs/data/就诊信息.csv。

#### 场景:下载模板
- **当** 用户在导入对话框点击"下载模板"链接
- **那么** 系统下载 docs/data/就诊信息.csv 文件，文件名显示为"就诊信息模板.csv"

### 需求:系统必须显示数据预览

系统必须在用户上传 CSV 文件后显示前 10 行数据预览，让用户确认导入内容。

#### 场景:显示预览
- **当** 用户上传 CSV 文件成功
- **那么** 系统显示数据预览表格，列头包括：CODE_ENT、NAME_PAT、CODE_PAT、DT_ACPT 等，最多显示 10 行数据

### 需求:系统必须在导入过程中显示进度

系统必须在导入过程中显示进度条和"正在导入..."状态文本。

#### 场景:显示导入进度
- **当** 用户点击"确认导入"按钮
- **那么** 系统显示进度条（indeterminate 模式）和"正在导入..."文本，禁用取消和导入按钮
